# 完成标准（Definition of Done - DoD）

> 任何任务在标记为 `[x]` 之前，必须逐项核对本清单。

---

## 代码质量

### 测试

- [ ] 所有测试通过（无失败）
- [ ] 测试覆盖率 ≥ 80%
- [ ] 核心函数至少一个测试用例
- [ ] 边界条件已测试
- [ ] 异常场景已测试

### 代码规范

- [ ] Lint 零错误
- [ ] 类型检查通过（TypeScript）
- [ ] 代码格式化（google-java-format / Prettier）
- [ ] 无编译警告

### 代码审查

- [ ] Code Review 已执行
- [ ] CRITICAL 问题已修复
- [ ] HIGH 问题已修复
- [ ] MEDIUM 问题已修复或记录到 TODO

---

## 架构约束

### 后端

- [ ] 遵循分层架构（Controller → Service → Mapper）
- [ ] 无跨层调用
- [ ] Controller 不含业务逻辑
- [ ] Service 不直接操作数据库
- [ ] Mapper 不含业务逻辑

### 前端

- [ ] 遵循组件化架构
- [ ] 状态通过 Pinia 管理
- [ ] API 调用统一封装在 `src/api/`
- [ ] 无直接操作 DOM

---

## 日志和异常

### 日志

- [ ] 关键操作有日志记录
- [ ] 日志包含上下文（user_id, request_id 等）
- [ ] 未使用 System.out.println / console.log
- [ ] 日志级别合理（DEBUG/INFO/WARN/ERROR）

### 异常处理

- [ ] 业务异常继承 BusinessException
- [ ] 异常在全局处理器统一处理
- [ ] 无空 catch 块
- [ ] 异常信息清晰

---

## 安全

- [ ] 无硬编码敏感信息（密码、密钥）
- [ ] 所有接口有权限校验
- [ ] SQL 使用参数化查询
- [ ] 用户输入已校验和过滤
- [ ] 密码使用 BCrypt 加密

---

## 文档

### 代码文档

- [ ] 公共方法有 JavaDoc / JSDoc
- [ ] 复杂逻辑有注释
- [ ] 注释使用中文

### 项目文档

- [ ] `docs/task-list.md` 状态已更新
- [ ] `CHANGELOG.md` 已更新
- [ ] API 文档已更新（如适用）
- [ ] 架构文档已更新（如适用）

---

## 性能

- [ ] 数据库查询有索引
- [ ] 无 N+1 查询问题
- [ ] 大列表有分页
- [ ] 慢查询已优化（< 1s）

---

## 可观测性

- [ ] 请求有 request_id 追踪
- [ ] 前端错误已捕获并上报
- [ ] 关键业务指标已埋点

---

## 部署

- [ ] 环境变量已配置
- [ ] Docker 镜像构建成功
- [ ] 部署文档已更新

---

## 自检清单

在提交前，请回答以下问题：

1. **代码是否可读？** - 其他人能否轻松理解？
2. **测试是否充分？** - 边界条件和异常场景是否覆盖？
3. **是否有技术债？** - 是否有 TODO/FIXME 需要处理？
4. **是否符合规范？** - AGENTS.md 和 CLAUDE.md 中的约束是否遵守？
5. **是否可以回滚？** - 如果出现问题，能否快速回滚？

---

**如果以上所有检查项都通过，任务可以标记为 `[x]` 完成。**
