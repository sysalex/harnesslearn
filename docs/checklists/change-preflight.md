# Change Preflight Checklist

在开始任何非只读任务前，先过这一份检查。

## 1. 任务分类

- [ ] 这是文档任务
- [ ] 这是配置任务
- [ ] 这是代码任务
- [ ] 这是 schema / API / 安全任务

如果同时命中多个分类，按最高风险处理。

## 2. 范围确认

- [ ] 已查阅 `docs/task-list.md`
- [ ] 当前任务已登记，或本次会补登记
- [ ] 已确认前置依赖
- [ ] 已确认不会误改无关区域

## 3. 风险扫描

- [ ] 是否影响 API 契约
- [ ] 是否影响数据库 schema / SQL
- [ ] 是否影响认证、授权、权限
- [ ] 是否影响 CI / Docker / 发布
- [ ] 是否影响关键日志、异常处理、可观测性

任一项为“是”，都要在动手前写清影响和验证方式。

## 4. 验证计划

- [ ] 明确要跑的测试或检查命令
- [ ] 明确是否需要安全审查
- [ ] 明确是否需要更新文档
- [ ] 明确是否需要更新 `CHANGELOG.md`

## 5. 交接计划

- [ ] 本次任务完成后会更新 `session-handoff.md`
- [ ] 如有经验或坑点，会更新 `.harness/learnings.md`

## 快速决策

- 文档类：可直接执行，但要同步 changelog / task 记录。
- 配置类：先确认影响范围，再执行，再验证。
- 代码类：遵守 TDD、DoD、审查要求。
- 高风险类：先确认，再执行。
