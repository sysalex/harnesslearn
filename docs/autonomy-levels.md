# Autonomy Levels

本文件定义 Agent 在本仓库中的默认自主级别，用于补充 `AGENTS.md` 中“禁止操作”和“需要确认”的规则。

## L0 - 只读分析

允许操作：
- 阅读代码和文档
- 搜索文件
- 分析问题
- 输出建议

不得操作：
- 修改任何文件
- 运行会改变状态的命令

## L1 - 低风险变更

允许操作：
- 修改文档
- 修改非生产配置
- 补充注释、规范说明、脚本说明

默认可直接执行，无需额外确认，但必须说明改动范围。

## L2 - 常规工程变更

允许操作：
- 普通代码改动
- 测试补充
- CI 配置调整
- 脚手架和目录结构补全

要求：
- 先查 `docs/task-list.md`
- 遵守 TDD / 验证 / 文档更新流程
- 任务完成且验证通过后，默认执行 `commit + push`，除非用户明确禁止
- 不得越过高风险边界

## L3 - 高风险变更

以下变更必须先确认：
- 删除文件或目录
- 修改数据库 schema 或初始化脚本
- 修改 API 契约
- 修改认证、授权、安全配置
- 修改发布、部署、生产环境相关配置
- 引入新的外部依赖

执行前必须说明：
- 改什么
- 风险点
- 回滚方式

## L4 - 禁止直接执行

以下操作默认禁止：
- 强制推送
- 跳过验证流程
- 删除测试以让构建通过
- 硬编码密钥或敏感信息
- 任何破坏性数据操作

## 使用方式

- 默认从 L0 开始判断任务。
- 能落在低级别，就不要抬到高级别。
- 一旦跨到 L3，必须显式确认。
