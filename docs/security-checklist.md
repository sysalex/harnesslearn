# 安全检查清单

> 对应 `docs/task-list.md` 任务 10.4。
> 每次发布前、重大功能合并前必须逐项核查。

## OWASP Top 10 检查项

### A01 访问控制失效
- [ ] 所有非公开 API 端点有 JWT 认证（Spring Security 配置）
- [ ] 角色权限检查到位（`@PreAuthorize` 注解）
- [ ] 员工只能查看自己的考勤记录，不能查看他人
- [ ] 部门主管只能查看本部门的考勤数据
- [ ] HR 可以查看全公司报表，但无法修改系统配置
- [ ] 敏感接口（如审批）有权限校验

### A02 加密失败
- [ ] 密码使用 BCrypt 哈希（cost factor ≥ 12），无明文存储
- [ ] JWT 密钥通过环境变量注入，不硬编码
- [ ] HTTPS 在生产环境强制启用
- [ ] 数据库连接字符串不出现在日志中

### A03 注入
- [ ] 所有数据库操作通过 MyBatis-Plus，无原生 SQL 拼接
- [ ] 用户输入经过参数校验（`@Valid` + `@NotBlank` 等）
- [ ] 无 SQL 注入风险（使用参数化查询）
- [ ] 无 XSS 风险（前端使用 Vue 模板，自动转义）

### A04 不安全设计
- [ ] 打卡状态机防止重复打卡
- [ ] 考勤规则变更需要 HR 权限
- [ ] 审批流程防止越权审批
- [ ] 批量操作有数量上限（防止 DoS）

### A05 安全配置错误
- [ ] 生产环境 `spring.profiles.active=prod`
- [ ] CORS 白名单仅包含已知域名，不使用 `*`
- [ ] 错误响应不暴露堆栈信息（全局异常处理器已处理）
- [ ] Swagger 文档在生产环境关闭或加认证

### A06 易受攻击和过时的组件
- [ ] Maven 依赖无高危漏洞（定期运行 `mvn dependency-check:check`）
- [ ] npm 依赖无高危漏洞（定期运行 `npm audit`）
- [ ] 定期更新依赖（每月检查一次）

### A07 身份认证和验证失败
- [ ] access_token 过期时间 ≤ 30 分钟
- [ ] refresh_token 过期时间 ≤ 7 天
- [ ] 登录失败不提示"用户名不存在"或"密码错误"（统一提示"用户名或密码错误"）
- [ ] 有登录失败频率限制（防止暴力破解）

### A08 软件和数据完整性失败
- [ ] 数据库迁移脚本经过 code review 后合并
- [ ] 生产环境迁移前有数据库备份
- [ ] SQL 脚本在版本控制中

### A09 安全日志和监控失败
- [ ] 登录成功/失败有日志记录（含 IP、时间）
- [ ] 权限拒绝有日志记录
- [ ] 打卡操作有日志记录（审计追踪）
- [ ] 日志不包含密码、token 等敏感字段

### A10 服务端请求伪造（SSRF）
- [ ] 无用户可控的 URL 请求（当前系统无此场景）

---

## 额外检查项

### 前端 XSS
- [ ] 无 `v-html` 使用（或使用时已 DOMPurify 净化）
- [ ] 无 `innerHTML` 直接赋值
- [ ] Element Plus 表单组件自带 XSS 防护

### CSRF
- [ ] 使用 JWT（无状态），天然防 CSRF
- [ ] 无 Cookie 存储 token（使用 localStorage）

### 敏感数据
- [ ] `.env` 文件在 `.gitignore` 中
- [ ] `git log` 中无密钥提交历史
- [ ] CI 中密钥通过环境变量注入

---

## 考勤系统特定检查项

### 打卡安全
- [ ] 打卡 IP 记录（防止代打卡）
- [ ] 打卡时间由服务端生成，不依赖前端
- [ ] 补卡申请需要审批，不能直接修改记录
- [ ] 考勤规则变更有审计日志

### 数据隐私
- [ ] 员工个人信息脱敏展示（如手机号、身份证）
- [ ] 考勤报表导出有权限控制
- [ ] 离职员工数据归档处理

---

## 检查频率

| 场景 | 频率 |
|------|------|
| 每次 PR 合并 | 核查 A01、A03、A09 |
| 每次发布 | 全量核查 |
| 每月 | 依赖安全扫描 |
